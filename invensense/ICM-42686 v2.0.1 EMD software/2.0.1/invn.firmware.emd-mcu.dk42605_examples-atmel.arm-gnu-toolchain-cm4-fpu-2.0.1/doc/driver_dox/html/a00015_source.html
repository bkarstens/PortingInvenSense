198  /* Accel Low Power could report with wrong ODR if internal counter for ODR changed overflowed
199  * WUOSC clock domain is informed through an 3bit counter that ODR
200  * has changed in RC/PLL mode. Every 8 event, this counter overflows
201  * and goes back to 0, same as 'no ODR changes', therefore previous ALP
202  * ODR is re-used.
203  * Solution: Keep track of ODR changes when WUOSC is disabled and perform dummy ODR changes when re-enabling WU after 8*n ODR changes.
204  */
205  uint32_t wu_off_acc_odr_changes;
